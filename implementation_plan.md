# Implementation Plan - Add "Developer" Settings Page and Centralized Logs

This plan outlines the architecture, layout, and implementation steps to add a **Developer** page inside the VS Code extension settings navigation, immediately after the **About** page. It also outlines the centralized logging integration across all modules using `LoggerService`.

---

## User Review Required

> [!IMPORTANT]
> The new Settings page will be placed immediately after the **About** tab in the vertical settings sidebar navigation. 
> To ensure maximum compatibility and performance, a virtualization engine (`VirtualList`) will be adapted for rendering very large lists of logs in real-time without blocking the main UI thread.

---

## Proposed Changes

We will register a new settings section `"developer"` in the extension webview settings, design the real-time logs dashboard (mirroring the styling and virtualization from the hub app), establish the IPC protocol for streaming developer logs, and instrument key packages to write logs to `loggerService`.

### 1. Extension Webview & UI Components

We need to add a new sidebar tab after "About", and render a developer log viewer with search, regex, filtering, export, and detailed JSON inspection.

#### [NEW] [developer-logs-view.tsx](file:///d:/V3/zenuxs-code/apps/vscode-extension/src/webview/components/DeveloperLogsView.tsx)
Create a new React component for the Developer Settings view, reusing the design, colors, icons, search filters, detail inspection pane, and export formats (JSON, CSV, TXT, Markdown) from `zenuxs-hub`. It will use a virtual list for performance.

#### [NEW] [virtual-list.tsx](file:///d:/V3/zenuxs-code/apps/vscode-extension/src/webview/components/VirtualList.tsx)
A performant React virtualized list component to render millions of logs smoothly.

#### [NEW] [json-view.tsx](file:///d:/V3/zenuxs-code/apps/vscode-extension/src/webview/components/JsonView.tsx)
A simple interactive JSON viewer for log details inspection.

#### [MODIFY] [SettingsView.tsx](file:///d:/V3/zenuxs-code/apps/vscode-extension/src/webview/components/SettingsView.tsx)
- Add `"developer"` to `SettingsSectionKey`.
- Import and render `<DeveloperLogsView />` when section is `"developer"`.
- Add `Developer` to `SIDEBAR_TABS` immediately after the `About` page.
- Use a code-like icon (terminal / code icon) for the tab.

---

### 2. IPC Communication Protocol

We will define new messages to synchronize logs and logging state between the extension host and the webview.

#### [MODIFY] [chat-view-provider.ts](file:///d:/V3/zenuxs-code/apps/vscode-extension/src/providers/chat-view-provider.ts)
- Add handler for `"developer_logs"` message.
- Support `"subscribe"`, `"unsubscribe"`, `"clear"`, `"pause"`, `"resume"` actions.
- Feed the log backlog to the webview immediately upon subscription.
- Stream new log events generated by `loggerService` to the webview.
- Forward `"webview_log"` messages from the webview to `loggerService`.

---

### 3. Log Instrumentation in Modules

We will instrument key modules to write trace and diagnostic events to `loggerService`.

#### [MODIFY] [handler-factory.ts](file:///d:/V3/zenuxs-code/packages/core/src/services/llms/handler-factory.ts)
- Wrap model clients returning from `createAgentModelFromConfig`.
- Log **LLM Requests**, **LLM Responses**, and **Streaming** details (first token, token counts, parser errors, completions, etc.).
- Log model definitions and settings under `devLogs.model.selected`.

#### [MODIFY] [strategy.ts](file:///d:/V3/zenuxs-code/packages/core/src/auth/strategy.ts)
- Log authentication flow milestones (token refresh, key auth, OAuth success/failures, retry attempts) using `devLogs.auth`.

#### [MODIFY] [sqlite-session-store.ts](file:///d:/V3/zenuxs-code/packages/core/src/services/storage/sqlite-session-store.ts)
- Log database read/write actions using `devLogs.storage`.

#### [MODIFY] [zenuxs-memory.ts](file:///d:/V3/zenuxs-code/packages/core/src/services/zenuxs-memory.ts)
- Log memory loading and sync events using `devLogs.memory`.

#### [MODIFY] [local-provider-service.ts](file:///d:/V3/zenuxs-code/packages/core/src/services/providers/local-provider-service.ts)
- Log API Key additions, updates, deletions using `devLogs.apiKey`.

#### [MODIFY] [extension.ts](file:///d:/V3/zenuxs-code/apps/vscode-extension/src/extension.ts)
- Intercept and mirror standard `console` functions (log, info, warn, error, debug) to the `loggerService` under the `console` category.
- Log extension lifecycle events (activation/deactivation).

### Manual Verification
- Open the settings panel in the extension.
- Verify the "Developer" tab appears directly after "About" with a correct icon.
- Click the "Developer" tab:
  - Live log entries stream in real-time.
  - Search/filter tools filter by provider, model, severity level, etc.
  - Clicking any log row opens the inspector panel showing JSON payload and metadata.
  - Try exporting (JSON, CSV, Markdown, TXT) and verify file download triggers.
  - Toggle Online/Offline status and verify network log creation.
  - Trigger LLM chat request and check that request, response, and stream logs appear in real-time.
